1.패러다임:스크립트언어,객체 지향 프로그래밍,

Javascript

: Javascript는 웹페이지에서 사용자 동작을
정해진 로직에 따라 처리하는 언어이다.
또한 객체기반의 스크립트 프로그래밍 언어로서,
이 언어는 웹브라우저 내에서 주로 사용되며
다른 응용프로그램의 내장 객체에도 접근할수 있는
기능을 가지고 있다.

Django

*특징*

1.구성: 장고는 파이썬으로 코딩한 모델을
관계형 데이터베이스로 구성 해주는
모델(Model), HTTP요청을 처리하는
웹 템플릿 시스템인 뷰(View)로 분리해
URL의 라우팅을 처리하는 URL 컨트롤러
로 구성된 MVC 디자인 패턴을 따른다.

2.모델: 모델은 데이터에 대한 정보를 담는다.
장고에서는 모델을 정의할때 필드의 종류를
지정해줘야 하는데, 이것이 데이터베이스에게
컬럼타입을 알려주고 HTML폼으로 표시될때의
입력타입도 내포하는 역할을 한다.

3.뷰: 어떤 데이터가 표시되는 지를 정의한다.
템플릿은 사용자에게 실제로 보여지는
웹페이지나 문서를 다룬다.

Django

: Django는 Python 기반의 네트워크 프레임워크로
데이터를 필요로 하거나, 서버의 자원을 필요로하는
사용자 요청을 서버에서 처리한다. 또한 보안이 우수하고 유지보수가 편리하며
무료 오픈소스 인데다가, 활발한 커뮤니티 들이 있고
좋은 참고자료와 무료및 유료지원을 하는 옵션들이 제공된다.

HTML

*개발*
: 1980년, 유럽 입자 물리 연구소의 계약자였었던 물리학자 팀 버너스리가 HTML의 원형인 인콰이어를 제안했다.
1989년에 팀 버너스리는 인터넷기반 하이퍼텍스트 체계를 제안하는 메모를 작성했다.
1990년 말에 팀 버너스리는 HTML을 명시하고, 브라우저와 서버 소프트웨어를 작성했다.

*속성*
: 대부분의 요소 속성들은 이름-값 형태이고, "="로 구분하고, 요소의 시작태그 안에서 요소의 이름 다음에 쓰인다.
값은 외따옴표 또는 쌍따옴표로 감쌀수있다.

*데이터형식*
: HTML은 스크립트 데이터와 스타일시트 데이터, 그리고 ID, 이름, URL, 숫자 길이와 단위, 언어, 색상, 문자
등을 포함하는 속성 값의 종류를 위한 몇 가지 데이터형태를 정의 내린다.
